# Engineering Rules

- Never guess.
- Verify current state before changing it.
- Request current files, screenshots or command output when state is uncertain.
- Deliver complete replacement files.
- Label files as replace, create or leave unchanged.
- One sprint has one goal.
- Verify everything.
- Keep documentation current after every confirmed change.
- Test before commit.
- Commit before push.
- Code and documentation are written in English.
- Development discussion is conducted in Polish.
- ChatGPT conversation titles are never technical identifiers.
- Repository writes require explicit user approval.
- No hidden or automatic paid operations.
